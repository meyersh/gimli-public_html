Description:

My Social Networking site (Copyleft ShaunCO) is built primarily using the MVC (Model, View, Controller) paradigm using much AJAX.

It's back-end is now 100% MySQL and accesses the database via a generalized wrapper aroundthe C-MySQL API provided from MySQL.com

The whiteboard (public + private) and messaging (public + private) systems all use the same messaging class I've generalized by providing an "entity" abstraction. Each entity, in essence, gets its own "inbox" with no practical limit to number of entities. All messages have a globally unique ID field by which they are referenced and allowed to REFER-TO for threading purposes.

Message access control utilizes a custom Blacklist module which maps directly to an Entity and allows creating/removing/checking-membership from a given entity. The only caveat here is since Randy directed the private whiteboard to allow NO-ONE by default (and my magick keywords---ALL and NONE---were not finished I simply directed the user to reverse the list...
